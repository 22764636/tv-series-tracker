Buttons

Solid buttons always pair bg-accent-solid (constant across themes) with text-white — never bg-accent with white text, since accent itself adapts per theme and isn't guaranteed to contrast with white.

Rating — two hearts (SeriesDetail + card)

RatingRow in SeriesDetail.jsx — only rendered when series.status === 'completed' (not for renewed). Two independent HeartRating rows, one per person: 💙 (series.ratingBlue) and 💜 (series.ratingPurple) — these two emoji are an explicit, one-off exception to the no-emoji rule, requested for this exact spot only. Each follows the same edit/display/add pattern as LinkRow: plain text "💙 X/10", a small text-xs text-muted "Modifica" button, or a "💙 + Aggiungi" text-accent hover:underline link when unset. Input: type="number" min="1" max="10" step="0.01" (up to 2 decimals), same input classes as the link field. Above the two heart rows, when at least one is set, a plain "Valutazione: X/10" line shows the averageRating() (progress.js) of whichever hearts have voted — if only one has, that single value stands in as the provisional average. formatRating() rounds to 2 decimals and strips trailing zeros (8 instead of 8.00). On the card, shown as "Voto X/10" using the same average — the status-badge row uses flex items-center justify-between so the rating sits flush right opposite the badge, instead of just trailing it.

Aggiorna da TMDB / Wikipedia links (SeriesDetail)

RefreshRow — a single text-sm text-accent hover:underline button ("Aggiorna da TMDB" / "Aggiornamento..." while pending), shown only when series.source === 'tmdb'. On error, an inline text-xs text-danger message next to the button — no modal, no toast. WikipediaRow — two plain text-accent hover:underline links, "Wikipedia (EN)" / "Wikipedia (IT)", same row, computed from the title on every render (never stored).

Sort control (Home)

Native <select> in rounded-lg border border-border bg-surface px-2.5 py-1.5 text-sm text-text shrink-0, same row as StatusTabs (parent: flex items-center gap-3, tabs wrapped in min-w-0 flex-1 so they can shrink/scroll instead of wrapping the select onto a second row). Options: ultimo aggiornamento (default), titolo (A-Z), valutazione, progresso — comparator lives in Home.jsx, reusing progressRatio from progress.js rather than re-deriving it.

StatusTabs (horizontal scroll on mobile)

Wrapper: no-scrollbar flex gap-2 overflow-x-auto; each pill button has shrink-0 added so it never squishes. On narrow screens where all status pills don't fit the row, it scrolls horizontally (touch/wheel) instead of wrapping to a second row and pushing the series grid down. .no-scrollbar (in src/index.css) hides the scrollbar track cross-browser while leaving scrolling itself enabled — used here and anywhere else a horizontally-scrollable row shouldn't show a visible track.

Header nav links

"Libreria" and "Calendario" — plain text-sm font-medium text-muted hover:text-text links, no pill/border, sitting left of the solid "+ Aggiungi serie" button. On screens below sm the title collapses to just "💙💜" (hidden/shown via sm:hidden / hidden sm:inline) so the whole header fits on one row instead of wrapping.

Calendario (month grid)

src/pages/Calendar.jsx — month-grid layout, mobile-first: 7 columns (Lun-Dom), each cell h-12 sm:h-14 rounded-lg border showing the date number plus one small dot (h-1.5 w-1.5 rounded-full) per scheduled/watched entry — bg-accent-solid for real watch history, bg-muted for future projected entries. Selected day: border-accent-solid bg-accent-soft. Today (unselected): border-accent-solid bg-surface. Tapping a cell selects it and shows its full entry list below in the same rounded-xl border border-border bg-surface px-3.5 py-2.5 row pattern used elsewhere; watched entries get an inline type="date" input to correct the recorded date. Future entries are derived live (never stored) from each series' watchDays + remaining unwatched count; past/today entries come from the real per-episode watched date — see src/lib/schedule.js. Navigation (← → buttons, mouse wheel, touch swipe — all three active at once) triggers a direction-aware slide+fade replay: the grid is remounted via a React key tied to ${year}-${month} with class animate-slide-next (from the right) or animate-slide-prev (from the left) depending on which way the month changed — keyframes in src/index.css. Wheel navigation is throttled with a short 150ms cooldown (a useRef lock, not React state) just enough to stop one trackpad gesture firing several months at once, without feeling laggy on deliberate fast scrolling.
